# Valuatorix — Comparable Companies Valuation

Machine learning valuation framework using Forbes Global 2000 companies as the comparable companies universe. Predicts EV/Revenue (and EV/EBITDA) multiples, finds the closest peers, and — for public companies — tells you whether the stock looks undervalued, overvalued, or fairly priced against a blended model/peer-median estimate. For private or manually-entered companies, with no market price to compare against, it shows a plain estimated valuation instead of a judgment.

**Live app:** `streamlit run app.py` — not yet deployed to a public URL (see D3 for status)

```bash
streamlit run app.py
```

The app trains models on startup (~2 seconds). If `data/forbes_enriched.pkl` exists, description-based TF-IDF matching is active.

```bash

```

Fetches business descriptions and financial metrics (EBITDA margin, gross margin, revenue growth) for all Forbes companies via Yahoo Finance. Runtime: ~10–20 minutes. Saves to `data/forbes_enriched.pkl`.

## Data

`data/forbes_global2000.xlsx` — Forbes Global 2000, 2026 edition (2,001 rows × 9 columns). Columns: Rank, Company, Country, Industry, Sales_B, Profits_B, Assets_B, MarketValue_B.

`data/forbes_enriched.pkl` — Generated by `enrich_forbes_descriptions.py`. Adds: Yahoo Finance ticker, business description, net debt (currency-converted to USD), revenue growth, EBITDA margin, gross margin, FCF margin. Coverage: ~84% of Forbes companies have a description; ~48% have financial-metric enrichment (EBITDA/gross margin, revenue growth); ~34% have usable EBITDA coverage for the EV/EBITDA model specifically.

**ML Tasks (app.py):**
- Regression: predict `ev_revenue` = EV ÷ Revenue (primary enterprise value multiple)
- Regression (secondary): predict `ev_ebitda` = EV ÷ EBITDA, trained on the subset of the pool with EBITDA coverage — shown as a parallel, lower-confidence metric, not blended into the primary estimate
- Classification: is the company trading below its industry-median EV/Revenue? (undervaluation flag)

## How the App Works

1. **Load a company** — Search by name or ticker (Yahoo Finance live lookup), or fill in financials manually in the sidebar and click "Load this company" to snapshot them the same way. Manual entry works for any company, including private ones not on Yahoo Finance. A currency-conversion fix ensures non-USD-reporting companies (e.g. Japanese, Korean tickers) show correct USD figures rather than raw local-currency values.

2. **Comparable-company scoring (5 factors)** — description similarity 50%, industry classification 20%, geography 15%, revenue size 10%, margin similarity (profit + EBITDA) 5%. The peer pool widens beyond the target's exact Forbes industry to related industries in the same cluster (e.g. Semiconductors for an IT Software target), with same-cluster peers scored down rather than excluded. Without a description, weights redistribute to industry 40% / geography 30% / revenue 20% / margin 10%.

3. **Model prediction** — two GradientBoosting regressors (trained on 1,901 Forbes companies after removing EV/Revenue outliers, log-transformed targets) predict fair EV/Revenue and EV/EBITDA multiples; a GradientBoosting classifier flags industry-relative undervaluation. Inputs: revenue, assets, profit, industry, revenue growth, EBITDA margin, gross margin.

4. **Verdict** — for public companies, compares the actual EV/Revenue against a blended benchmark (30% the model's own prediction, 70% the peer median) — ≥13% below = undervalued; ≥13% above = overvalued. For private/manually-entered companies, with no market price to compare against, the app shows the blended figure as a plain "Estimated Valuation" instead of an undervalued/overvalued/fair-value judgment.

5. **Display units** — a sidebar toggle switches every dollar figure (manual inputs, tiles, comps table, charts) between Billions/Millions/Thousands; already-entered values convert live.

6. **Export** — download the comps table as CSV, or the full report (company overview, valuation, verdict, comps table) as a multi-sheet Excel workbook.

## ML Pipeline

- **Training data:** 1,901 Forbes Global 2000 companies (of 1,999 with usable Sales_B), after filtering to EV/Revenue between 0.01× and 200×, across 27 industries. The EV/EBITDA regressor trains on the ~34% subset with usable EBITDA coverage.
- **Features:** Revenue, Assets, Profits, profit margin, log(Revenue), log(Assets), Industry (OHE) + EBITDA margin, gross margin, revenue growth (when available) — identical feature set for both regressors and the classifier
- **Feature selection:** `analysis.qmd` (D1) applies Recursive Feature Elimination (RFE) inside the leakage-safe pipeline, keeping the top half of transformed features for each task — report-only, `app.py`'s live models use the full feature set
- **Target:** log1p(EV/Revenue) and log1p(EV/EBITDA) for the two regressors — log-transform reduces right-skew and cuts overfitting (EV/Revenue CV R² 0.10 → 0.45, overfitting gap 0.70 → 0.11)
- **Preprocessing:** Median imputation + StandardScaler for numerics, OHE for industry
- **Financial sector treatment:** Banks, Insurance, Diversified Financials — net debt set to zero (customer deposits are not economic debt); EV = Market Cap only
- **Currency handling:** dollar-denominated fields are converted to USD using the ticker's reported currency before any computation — market cap uses the trading currency, financial-statement fields use the (possibly different) reporting currency, since these can diverge for ADRs
- **EV/Revenue & EV/EBITDA:** EV = Market Cap + Net Debt; sourced from Yahoo Finance via `enrich_forbes_descriptions.py`
